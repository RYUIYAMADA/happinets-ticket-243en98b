# SPEC.md — 選手家族チケット申込システム
作成日: 2026-05-07 | PM裁定

## 目標
選手家族チケットの申込・確認・管理をWeb完結させ、チケット担当への無駄な問い合わせをゼロにする。

## 成功基準（検証可能）
- [ ] 選手が自分のID（選手番号）でログインし、3種類のチケットを試合ごとに申込できる
- [ ] 申込フォームに試合ごとの申込期限が表示され、期限超過時は申込不可になる
- [ ] 選手ダッシュボードで全試合の申込状況（未申込/申込中/確保済/締切）が一覧確認できる
- [ ] 管理者が各試合の申込期限を設定・変更できる
- [ ] 管理者が試合別・選手別の申込一覧を確認し、確保状況を更新できる
- [ ] 全データがGoogleスプレッドシートに自動記録される

## 非目標（スコープ外）
- 決済機能（有料チケットは申込のみ、支払いは別途）
- メール通知自動送信（Phase2以降）
- スマホアプリ化
- 選手の座席番号指定

## 技術スタック（PM裁定）
| レイヤー | 技術 | 理由 |
|---|---|---|
| フロントエンド | 静的HTML/CSS/JS（単一ファイル構成） | デプロイ不要・即使用可 |
| バックエンド | Google Apps Script Web App | 無料・Spreadsheet直接連携 |
| DB | Google Spreadsheet | 管理者がExcelライクに確認可能 |
| 認証 | 選手番号（4桁）+ パスワード（簡易） | 内部ツールのため軽量認証で十分 |

## 画面構成（4画面）

### 1. ログイン画面 (`index.html`)
- 選手番号 + パスワード入力
- 管理者ログイン（別パスワード）

### 2. 選手申込フォーム (`player-form.html`)
- 試合選択（期限内のみ選択可）
- チケット種別選択（招待/家族席/有料）
- 種別ごとのフィールド表示（条件分岐）
- 申込確認 → 送信

#### 招待チケットフィールド
- 試合選択（必須）
- 招待枚数（1〜4枚）
- 受取者氏名
- 受取方法（事前受取/当日受取）
- 備考（任意）

#### 家族席チケットフィールド
- 試合選択（必須）
- 枚数（大人/子供/未就学児 各枚数）
- 受取者氏名
- 受取方法（事前受取/当日受取）
- 備考（任意）

#### 有料チケットフィールド
- 試合選択（必須）
- 席種（S席/A席/B席）
- 枚数（1〜4枚）
- 受取者氏名
- 受取方法（事前受取/当日受取）
- 備考（任意）

### 3. 選手ダッシュボード (`player-dashboard.html`)
- 全30試合の申込状況一覧テーブル
- ステータス: 未申込（灰）/ 申込中（黄）/ 確保済（緑）/ 締切（赤）
- 各試合の申込詳細確認（クリックで展開）
- 申込修正・キャンセルボタン（期限内のみ）

### 4. 管理者画面 (`admin.html`)
- 試合一覧 + 各試合の申込期限設定
- 選手別申込状況テーブル（試合×選手のマトリクス）
- 申込詳細確認・確保状況更新（pending→confirmed/rejected）
- CSVエクスポート

## Spreadsheetシート構成
| シート名 | 内容 |
|---|---|
| `players` | 選手ID・氏名・パスワードハッシュ |
| `games` | 試合ID・日付・曜日・対戦相手・申込期限 |
| `applications` | 申込ID・選手ID・試合ID・種別・詳細JSON・申込日時・ステータス |
| `settings` | システム設定（管理者パスワード等） |

## Gate1 修正指示（PM裁定 2026-05-07）

### 技術（senior-engineer concern → 対応済）
1. **CORS**: GASレスポンスは `ContentService.createTextOutput(JSON.stringify(result)).setMimeType(ContentService.MimeType.JSON)` で返す。fetchは `redirect: 'follow'` を指定
2. **認証**: クライアント側でSubtleCrypto SHA-256ハッシュ化 → GASに送信。Spreadsheetにはハッシュのみ保存
3. **applications シート**: 詳細JSONではなくカラム分割（ticket_type, quantity_adult, quantity_child, quantity_infant, seat_type, receiver_name, pickup_method, note）

### UI（senior-designer concern → 対応済）
1. **フォーム種別切替**: ラジオカード縦積み（幅100%）→選択後に該当フィールドを表示切替
2. **ダッシュボード試合一覧**: テーブル禁止 → カードリスト型（試合名+日付+ステータスバッジ右端）
3. **ステータスカラー**: 申込中は背景`#FEF3C7`文字`#1A1A2E`でコントラスト確保
4. **タッチターゲット**: 最小44×44px厳守
5. **日本語禁則**: `line-break: strict` 全画面適用
