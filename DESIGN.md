# DESIGN.md — family-tickets（選手家族チケット申込システム）

## 1. このプロダクトは何か

- **何をするものか**: ハピネッツ選手の家族が試合チケットを申し込む専用フォームと、選手・運営それぞれが申込状況を確認・管理する画面群。
- **主な利用者**: 選手の家族（IT操作に不慣れな層を含む）、選手本人、ハピネッツ運営スタッフ
- **利用デバイス/環境**: スマートフォン中心（iOS Safari / Android Chrome）。PCでの管理画面利用あり。
- **言語**: 日本語・英語の2言語対応（player-form.html / player-dashboard.html）
- **トーン**: 親しみやすく・安心感がある。余白を広めに取りストレスなく読める。ハピネッツブランドカラー（黒・ピンク・ゴールド）を一貫して使う。

---

## 2. デザイン原則

1. **読む負担ゼロ** — 非IT層の家族が迷わず操作できること。テキスト量・選択肢・ボタン数を最小化し、1画面1タスクを基本とする。
2. **ブランド一貫性** — 4画面すべてで同一のカラートークンを使う。画面ごとに色が変わると信頼感が損なわれる。
3. **タップ優先** — スマホユーザーがメイン。タップターゲット 44px 以上、項目間の余白は 8px グリッドで確保する。
4. **アクセシビリティは義務** — 家族層は高齢者を含む可能性がある。コントラスト・フォントサイズ・フォーカス表示は妥協しない。
5. **変数で統治** — CSS 変数を一箇所変えれば全画面に反映される状態を保つ。ハードコードは禁止。

## 3. カラートークン（4画面共通・正とする値）

4画面（index / player-form / player-dashboard / admin）の `:root` を照合した結果。
差異があった変数は「正とする値」を定め、全画面で統一する。

```css
:root {
  /* ── ブランドカラー ── */
  --primary:          #000000;   /* 用途: メインアクション・ヘッダー */
  --primary-hover:    #1a1a1a;   /* 用途: --primary のホバー状態 */
  --accent:           #EC008C;   /* 用途: CTA・強調・ハピネッツピンク */
  --accent-hover:     #c4006e;   /* 用途: --accent のホバー状態 */
  --gold:             #E6B422;   /* 用途: バッジ・ゴールドハイライト */
  --gold-hover:       #c9971b;   /* 用途: --gold のホバー状態 */

  /* ── 背景・サーフェス ── */
  --bg:               #ffffff;   /* 用途: ページ背景 */
  --card:             #ffffff;   /* 用途: カード背景（※）*/
  --bg-highlight:     #fff5ec;   /* 用途: 注目エリア背景（player-form/admin使用）*/
  --bg-soft:          #fff8f3;   /* 用途: ソフトな強調背景 */
  --bg-muted:         #f1f5f9;   /* 用途: 非アクティブ・グレーアウト背景 */
  --bg-surface:       #f8fafc;   /* 用途: セクション区切りなど薄いサーフェス */

  /* ── テキスト ── */
  --text:             #1a202c;   /* 用途: 本文・主テキスト */
  --text-sub:         #4a5568;   /* 用途: 補足テキスト・ラベル（※）*/
  --muted:            #6b7280;   /* 用途: プレースホルダー・無効テキスト */

  /* ── ボーダー・ライン ── */
  --border:           #e5e7eb;   /* 用途: 区切り線・入力枠 */

  /* ── ステータス ── */
  --success:          #16a34a;   /* 用途: 完了・成功 */
  --success-bg:       #dcfce7;   /* 用途: 成功バナー背景 */
  --error:            #dc2626;   /* 用途: エラー・必須未入力 */
  --error-bg:         #fee2e2;   /* 用途: エラーバナー背景 */
  --warning-bg:       #FEF3C7;   /* 用途: 注意バナー背景 */
  --warning-text:     #1A1A2E;   /* 用途: 注意バナー文字色 */
  --warn:             #fff8e1;   /* 用途: 警告アイコン周辺の背景（player-form使用）*/
  --warn-border:      #f59e0b;   /* 用途: 警告枠の border-color */
  --muted-bg:         #f3f4f6;   /* 用途: 無効ボタン・非選択タブ背景 */

  /* ── 形状 ── */
  --radius:           6px;       /* 用途: カード・ボタン・入力欄の角丸（全統一）*/
}
```

| 変数 | index | player-form | player-dashboard | admin | 正とする値 |
|---|---|---|---|---|---|
| `--card` | `#ffffff` | `#fff` | `#ffffff` | `#fff` | `#ffffff`（6桁で統一）|
| `--text-sub` | `#374151` | `#4a5568` | `#4a5568` | `#4a5568` | `#4a5568`（3画面多数派）|
| `--bg-highlight` 〜 `--warn-border` | 未定義 | 一部定義 | 一部定義 | 一部定義 | 全画面で定義（上記トークン表が正）|

## 4. タイポグラフィ

```css
--font-base:    'Noto Sans JP', sans-serif;
--font-heading: 'Noto Sans JP', sans-serif;
```

| 用途 | font-size | font-weight | 備考 |
|---|---|---|---|
| ページ見出し（h1） | 1.5rem〜2rem | 700 | |
| セクション見出し（h2/h3） | 1.1rem〜1.3rem | 700 | |
| 本文 | 1rem（16px） | 400 | 最小サイズ |
| ラベル・補足 | 0.875rem | 400〜500 | 小さすぎる場合は本文サイズに統一 |
| ボタン | 0.9rem〜1rem | 600〜700 | |

- font-weight 300 以下の使用禁止（細すぎてモバイルで読めない）
- 日本語テキストは文節改行・禁則処理を適用（`overflow-wrap: break-word; word-break: keep-all;`）

---

## 5. レイアウト規約

- **最大幅**: フォーム・カード系は `max-width: 640px`、管理画面は `max-width: 1200px`
- **グリッド**: 8px グリッド（`--space-1: 8px` 単位で余白設計）
- **ブレークポイント**: `768px`（スマホ/PC の境界）
- **情報密度**: ゆったり（スマホ家族層がメイン。詰め込まない。フォームは1問ずつ視認しやすく）

---

## 6. 禁止ルール（anti-pattern・最重要）

| 禁止 | 理由 |
|---|---|
| **CSS 値のハードコード**（`color: #EC008C;` など直書き） | トークン変数が崩壊する。変数 `var(--accent)` を使う |
| **グラデーション** | ハピネッツブランドはフラットデザイン。グラデーションは安っぽく見える |
| **glassmorphism（backdrop-filter: blur 等）** | モバイル描画コストが高い。古く見える |
| **カードに `border` を付ける** | `box-shadow` を使う。border は情報密度を上げすぎる |
| **円グラフ** | 数値比較に不向き。棒グラフ・数値カードを使う |
| **font-weight 300 以下** | スマホ・高齢者層で視認性が著しく低下する |
| **1画面に primary ボタンを複数配置** | ユーザーが迷う。主アクションは必ず1つに絞る |
| **`--text-sub` に `#374151`（index 旧値）を使う** | 正値は `#4a5568`。旧値は削除して統一する |

---

## 7. インタラクション規範

- **タップターゲット**: 最小 44×44px（ボタン・チェックボックス・リンクすべて）
- **フォーカス表示**: `:focus-visible` で明示的なアウトラインを必須とする（ブラウザデフォルトを消さない）
- **コントラスト比**: テキスト 4.5:1 以上 / UI 要素・アイコン 3:1 以上（WCAG 2.1 AA）
- **エラー表示**: `--error` / `--error-bg` を使い、色だけでなく文字でもエラー内容を示す

---

## 8. Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| `color: var(--accent);` でブランドピンクを適用する | `color: #EC008C;` とハードコードする（変数が変わっても追従できない）|
| カードに `box-shadow: 0 2px 8px rgba(0,0,0,.06);` を使う | カードに `border: 1px solid var(--border);` を使う（情報が詰まって見える）|
| ボタンラベルは「申し込む」「確認する」など動詞で完結させる | ボタンラベルを「こちら」「OK」など曖昧な表現にする（非IT層が迷う）|
| フォームエラーは `--error-bg` + `--error` + 日本語メッセージで表示する | エラーを色だけ（赤枠）で示す（色覚特性のあるユーザーが判別できない）|
| `font-weight: 700` で見出しに強弱をつける | `font-weight: 300` で細い装飾テキストを使う（スマホで読めない）|

## 9. AI（Claude/Codex）への指示

- UI 実装前に必ずこのファイルとグローバル DS を読む
